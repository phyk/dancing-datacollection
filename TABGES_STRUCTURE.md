# tabges.html Structure and Interpretation

This document describes the structure and common headers found in `tabges.html` files generated by TopTurnier, used for both DTV-native and WDSF competition results.

## Overview

The `tabges.html` (Total Table / Wertungstabelle Gesamt) provides a condensed view of a competition's progress across all rounds. It uses a grid-like structure where rows represent different data points (Judges, Results, Qualifications) and columns typically correspond to Bib numbers (Startnummern).

## Metadata Headers

These are usually found at the top of the file or within the first few rows of a table.

| German Header | English Header | Meaning | Model Mapping |
| :--- | :--- | :--- | :--- |
| **Anzahl Teilnehmer** | **No. of participants** | Total count of competitors in the event. | `Competition.participants.len()` |
| **Startnummer** | **Start number** | Column anchor for participant bib numbers. | `Participant.bib_number` |
| **Wertungsrichter** | **Adjudicators** | Row identifying the judge codes/names. | `Officials.judges` |

## Round Headers

Each round (Vorrunde, Zwischenrunde, Endrunde/Final) is demarcated by a set of rows.

| German Label | English Label | Description | Model Mapping |
| :--- | :--- | :--- | :--- |
| **Ergebnis der [X]. Runde** | **Result of [X] round** | The sum of marks or crosses for that round. | `RoundData::Marking` |
| **Qualifiziert für die ...** | **Qualified for ...** | Indicates if the participant advanced (often 'X'). | Round Progression Heuristic |
| **Addition Endrunde** | **Result of final** | Sum of ranks or skating points in the final. | `RoundData::DTV` / `RoundData::WDSF` |

## Judge and Score Interpretation

### Judge Markers
Judges are identified by abbreviations (e.g., `AE`, `AW` in DTV; `A`, `B` in WDSF).
- In **Vertical** layouts, these codes are listed in the first column of each round block.
- In **Horizontal** layouts (less common in `tabges`), they may appear as column headers.

### Scoring Types
1.  **Marking Round (X / -):** Represented by cross counts or numeric sums of crosses. Mapped to `RoundData::Marking`.
2.  **DTV Final (Skating):** Numeric ranks (1-6) from each judge. Mapped to `RoundData::DTV`.
3.  **WDSF Absolute Judging:** Decimals (e.g., `35.20`) often with component labels like `TQ`, `MM`, `PS`, `CP`. Mapped to `RoundData::WDSF`.

## Final Results Section

Located at the bottom of the table.

| German Label | English Label | Description | Model Mapping |
| :--- | :--- | :--- | :--- |
| **Platz von / Platz bis** | **Rank from / Rank to** | The final ranking range (e.g., 7.-8.). | `Participant.final_rank` |
| **Aufstiegspunkte** | **Promotion points** | Points earned towards the next level (DTV). | (Not currently in model) |
| **Class** | **Class** | Participant's level (e.g., S, A). | `Competition.level` |

## Common HTML Patterns

- **Table Class:** `tab1` is the standard class for the result tables.
- **Cell Classes:**
  - `td1`: Header labels / Bold keys.
  - `td2`: Metadata values.
  - `td3`: Data labels (e.g., "Result of...").
  - `td5c` / `td5cv`: Result cells containing marks or ranks.
  - `td2gc`: Bib number headers with tooltip span for names.

## Commonalities vs Differences

### Commonalities
- Always follows a "Bib-Anchored" column strategy.
- Uses `<br>` tags within cells to separate data from different judges or components.
- Tooltips (`span.tooltip2gc`) are used to link Bib numbers to full names in the header.

### Differences
- **Language:** DTV-native events are primarily German; WDSF events are English.
- **Granularity:** `tabges.html` is often a summary. If granular dance-level marks are missing, they must be retrieved from `ergwert.htm` or `erg.htm`.
- **Scoring System:** DTV uses integers (ranks/crosses); WDSF uses floats (scores).
